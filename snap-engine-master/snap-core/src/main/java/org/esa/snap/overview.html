This is the overview page of the BEAM Java API and examples documentation.

Overview

The BEAM (Meris and (A)ATSR ToolBoX) Java library comprises a development framework and a great number of packages containing specific, problem-oriented implementations.

The BEAM Framework API

The BEAM Framework API is a core part of the BEAM API which defines all interfaces, classes and recurring design patterns used to implement concrete software modules such as the processing tools, data product readers and writers or map projections.

This package provides the software frameworks used throughout the MERIS/(A)ATSR Toolbox software.

Currently, the following framework packages exist:

  • {@link org.esa.snap.core.datamodel} - Product in-memory resentation - the most important BEAM package
  • {@link org.esa.snap.core.dataio} - Product Data I/O plug-in framework
  • {@link org.esa.snap.core.dataop} - Product data manipulation interfaces and design patterns
  • {@link org.esa.snap.core.processor} Product processing framework, used by all scientific data processors
  • {@link org.esa.snap.core.param} - Generalized parameter handling
  • {@link org.esa.snap.core.ui} - User interface (UI) related classes and sub-packages

Note that all classes and packages contained in org.esa.snap.framework are independent from other org.esa.snap.* classes and packages.

The {@link org.esa.snap.util} Package

This package provides several utility classes and sub-packages used throughout the MERIS/(A)ATSR Toolbox software.

Currently the following sub-packages exist:

  • {@link org.esa.snap.util.io} - I/O-Utilities
  • {@link org.esa.snap.util.jai} - Support classes for JAI (Java Advanced Imaging API)
  • {@link org.esa.snap.util.math} - Mathematical helper classes
  • {@link org.esa.snap.util.logging} - Logging capabilities

The BEAM API Source Code Examples

Provided here is a short documentation for the Java source code examples provided along with the BEAM Software. It shows the basic concepts on how to use the BEAM Java API for your own applications:

1. {@link org.esa.snap.examples.data_export}: very simple examples showing how to read data from a product and write them out: The package conatins three source files:

  • BandWriterMain.java
  • BitmaskWriterMain.java
  • TiePointGridWriterMain.java

2. {@link org.esa.snap.examples.ndvi_simple}: a very simple NDVI (normalized differential vegetation index) processor. The package contains a single source file:

  • NdviSimpleMain.java

3. A more advanced NDVI processor which uses parts of the BEAM processing framework can be found on the BEAM plug-in page.

In order to compile the examples, the ZIP file also contains a build.xml file for use with the open source tool ant (a kind of make for Java) developed by the Apache Software Foundation.

Newcomers who want to develop export tools and data processors are invited to look at {@link org.esa.snap.examples.data_export} and {@link org.esa.snap.examples.ndvi_simple} first. The first package contains basic tools which lets you extract geophysical data from products. The {@link org.esa.snap.examples.ndvi_simple} package contains a few lines of code showing you how to compute the NDVI value from a MERIS Level 1b input product.

Once you have understood that, please examine the {@link org.esa.snap.examples.ndvi_processor} example contained in this package. {@link org.esa.snap.examples.ndvi_processor.NdviProcessorMain} implements is a fully functional data processor which uses some important concepts and mechanisms of the BEAM processing framework. Don't be afraid, it is still very easy code. It can be used as a template for your own data processors.

This is the list of the most important API packages used by the examples:

  1. The {@link org.esa.snap.core.datamodel} package.
  2. The {@link org.esa.snap.core.dataio} package.
  3. The {@link org.esa.snap.util} package.
